GraphQl Day-3 Real-Time Data

Description

1. Integrate a WebSocket library (e.g., subscriptions-transport-ws) into your GraphQL server.  Configure WebSocket connections to work seamlessly with GraphQL subscriptions.

2.Extend the existing GraphQL schema to include a subscription for new comments.Implement the corresponding resolver to handle real-time updates when new comments are added.

3.Extend the GraphQL schema to include types like User and Message for the chat application.Implement mutations to send messages and handle user authentication.

4.Extend the existing GraphQL schema to include user presence information.Implement subscriptions to notify clients about users joining or leaving the chat.

5.Extend the GraphQL schema to include a query for fetching message history.Implement a resolver to handle requests for retrieving message history.

Here are suggestions for tackling Assignment 3 (GraphQL Real-Time Data):

1. **WebSocket Integration**  
   - Use libraries like `subscriptions-transport-ws` or `graphql-ws` for WebSocket support.
   - Update your GraphQL server setup to enable subscriptions (e.g., with Apollo Server, configure the subscription server).

2. **Subscription for New Comments**  
   - Add a `Subscription` type in your schema, e.g., `newComment`.
   - Use a PubSub mechanism (like Apollo's `PubSub`) to publish new comments.
   - Implement a resolver for the subscription that pushes updates when a comment is added.

3. **User and Message Types, Mutations, Authentication**  
   - Define `User` and `Message` types in your schema.
   - Add mutations for sending messages and user login/signup.
   - Implement authentication (JWT or session-based) in your resolvers.

4. **User Presence and Subscriptions**  
   - Add fields for user presence (e.g., `isOnline`) in your schema.
   - Implement subscriptions to notify clients when users join/leave.
   - Track user connections/disconnections in your server logic.

5. **Message History Query**  
   - Add a query to fetch message history (e.g., `messages`).
   - Implement a resolver that returns messages from your data source (DB or in-memory).

**General Tips:**  
- Use Apollo Server or similar frameworks for easy subscription setup.
- Test subscriptions with GraphQL Playground or Apollo Client.
- Modularize your schema and resolvers for maintainability.
